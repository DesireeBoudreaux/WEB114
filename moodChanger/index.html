JavaScript Q&As

Why is the JavaScript textContent property safer than innerHTML?

textContent inserts only plain text. It doesn't run HTML or JavaScript, avoiding malicious code and scripts.

If use an HTML text box on a page and it has an attribute (for example id="text") and a corresponding DOM property for example, const box = document.getElementById("text"). How do the two elements relate?

id identifies the element in HTML. getElementById gives a JavaScript reference to that element.

What would happen to the text box with id="text" if it added this line to the code above: text.textContent = "Hello!"?

The element with id="text" will display the message Hello!
